信用研究 AI Analyst

  • 企业信用研究 AI Analyst 是固收团队基于可复用技能（Skill）框架开发的信用研究 Agent。团队将基础规则（数据提取逻辑、计算公式、报告模板）、方法论（信用评级方法论）、决策逻辑（实际工作中累积的经验与判断标准）等知识和经验结构化沉淀，写入 Agent 的 Skill 体系。这一过程实现了从"人的经验"到"机器可理解的知识"的转化。
  • 在此基础上，团队构建了可由 Agent 自主执行的信用评级工作流程，使 Agent 独立完成从信息收集到报告交付的全流程工作。
  • 以标准测试项目为例，Agent 在约 1.5 小时的有效处理时间内，协同多个专业子代理检索、解析、核验并整理 60 余份各类研究资料，完成 8 个研究阶段。
  • 展望将来，通过持续的"反馈学习"，信用研究 Agent 将不断迭代成长，提升对复杂案例中的分析和判断能力，并将每一次学习成果和研究报告沉淀为组织永久性的知识资产。
"AI Analyst 作为团队的一部分，帮助放大团队的能力，使团队将精力聚焦于高价值的深度分析与投资决策"

信用研究 AI Analyst —— Agent 设计

  • 信用研究 AI Analyst 是由 Skill 驱动的工具增强型 Agent。Agent 依据 Skill 定义的规则与流程分解任务、调用工具，并通过 Agentic Loop（任务规划—工具调用—结果校验）完成信用研究工作。

信用研究 AI Analyst —— 基于 Skill 的 Agent 工作流

  • Skill 将信用研究拆成多个 Task。每个 Task 都规定了目标、步骤、输入和输出，Agent 在该边界内反复完成“读取资料 → 推理规划 → 调用工具 → 检查修正”，直到阶段结果满足要求。
  • 每个 Task 都由 Skill 定义边界，并在边界内循环执行。每个 Task 的输出会沉淀为下一阶段上下文，在保留关键结论的同时压缩原始资料占用，使 Agent 能稳定执行长链条研究任务。
  • Task 化设计符合信用评级的专业流程，让主体识别、研究、评分和评级推导逐步推进，并支持阶段性复核。

Agent 产品/工具与底座模型对比

  • 产品与工具选型重点考察 Skill 兼容性、长文档处理、工具与桌面应用接入、权限治理及过程可追溯性，并需进行业务适配与回归测试。
  • 信用研究以资料处理、跨应用协作和文档交付为主，Work 类 Agent 比 Code Agent 更匹配；综合能力与当前生态，优先选择 ChatGPT Work。

问题与实践经验 1 —— 引入 subagent

  • 创建 subagent，专门处理输入数据量大、噪声高、输出可压缩的任务。subagent 负责专业小项目，并只返回可引用的摘要结果。
  • 以同业研究为例，通常涉及被研究对象以外同业可比公司大量年报、公告、投资者材料、评级报告和行业数据，但我们所需要的是高度精炼少量结论。我们用 peer_company_researcher 启动 subagent，subagent 每次只研究一家可比公司，重点输出核心事实、结论、关键证据。这样，主 Agent 不再直接处理全部原始材料，不会“lost in the middle”。

问题与实践经验 2 —— 阶段性摘要节点和主动上下文压缩

  • 对于复杂任务，即便通过"Task 任务分解"，仍不能很好地解决"lost-in-the-middle"问题。
  • 在每个 Task 里，我们通过设置"阶段性摘要节点"确保 Agent 输出达到质量要求，并通过主动上下文压缩控制上下文长度。
【阶段性摘要节点】
每完成一个章节（如公司概况、业务分析、财务分析），自动生成结构化摘要（含文字分析、数据表格），存入向量数据库。
【恢复锚点】
项目根目录维护 CURRENT_RATING_STATE.md：只记录当前进度、当前/下一步 prompt 路径与已完成输出路径的最小导航信息。属恢复用途、非交付物，不列为资料来源。
自动上下文压缩：主任务保留关键摘要，subagent 输出结构化结论
21

问题与实践经验 3 —— few-shot 提示词

  • 当输入指令不够清晰时，Agent 随机尝试不同方案，导致产出不稳定的结论。
  • 通过重复测试，我们发现重复出错的地方，补充更明确的 prompt。
  • 以关键信息提取为例，我们给 Agent 明确的使用工具要求，保证每次都能提取到完整信息。

以目录结构做状态管理，Task 结果落盘为阶段交付物，将上下文从模型短期记忆迁移至外部存储

数据溯源与缺失管控
关键数据保留来源与路径；缺失数据不编造、不补齐，必须标记并说明影响
